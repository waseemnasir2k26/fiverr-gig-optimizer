What Is This?

The Fiverr Gig Optimizer is a free, open-source tool that uses Claude AI to generate a complete Fiverr gig strategy for you. Instead of guessing at titles, descriptions, and pricing, you tell Claude your services and it:

  1. Researches Fiverr competition for your keywords
  2. Finds combo-niche gaps — keyword combinations with 10-100x less competition
  3. Generates optimized gig titles under 80 characters
  4. Writes high-converting descriptions with cross-sell callouts
  5. Creates professional thumbnails with your photo (1280x769px)
  6. Designs 3-tier pricing (Basic / Standard / Premium)
  7. Plans a phase-based rollout so you don't overwhelm yourself
  8. Maps cross-sell funnels so every gig leads to the next
The Result A single HTML file you open in your browser with everything copy-paste ready. Download thumbnails, copy titles, descriptions, and tags — then create your gigs on Fiverr in minutes.

0 Prerequisites

Before starting, make sure you have:

  • Claude Code — Install from claude.ai/code (free tier works)
  • Python 3.8+ — Check with python --version
  • Git — Check with git --version
  • A professional photo — Headshot, PNG or JPG, minimum 500x500px. Photos in thumbnails get 25-40% higher click rates.
  • A Fiverr account — Free to create at fiverr.com
No Photo? No Problem The tool works without a photo too — it'll generate thumbnails with a stylish gradient placeholder instead. But we strongly recommend adding your face for better CTR.

1 Clone the Repository

Open your terminal and run:

git clone https://github.com/waseemnasir2k26/fiverr-gig-optimizer.git
cd fiverr-gig-optimizer

Place your professional headshot photo in this folder. Name it photo.png (or any name — you'll tell Claude where it is).

2 Run the Optimizer

Open Claude Code in the repo directory:

claude

Then type the slash command:

/fiverr-optimize

Claude will ask you a series of questions:

Question Example Answer
Your name Jane Smith
Brand name (optional) Digital Spark Agency
Website (optional) digitalspark.io
Your services n8n automation, WordPress sites, AI chatbots, GoHighLevel CRM, voice agents, SEO
Photo path photo.png
Current Fiverr gigs fiverr.com/my-gig-1, fiverr.com/my-gig-2 (or "none")
Monthly goal $5,000
Seller level New seller
Pro Tip: List EVERY Skill The more services you list, the more combo-niche opportunities Claude can find. "WordPress + AI" has way less competition than just "WordPress." Don't hold back — list everything.

3 Review Your Strategy

Claude generates two files:

  1. gig-config.json — All your gig data (titles, descriptions, tags, pricing, thumbnail settings)
  2. fiverr-catalog.html — The visual catalog with everything rendered

Open the catalog in your browser:

# Windows
start fiverr-catalog.html

# Mac
open fiverr-catalog.html

# Linux
xdg-open fiverr-catalog.html

You'll see a dark, professional dashboard with:

  • Phase-based gig cards — Each gig with its thumbnail, title, description, tags, and pricing
  • Cross-sell funnel diagram — Visual map of how gigs connect
  • Competition data — How many competitors each gig faces
  • Action plan — Week-by-week checklist

4 Create Your Fiverr Gigs

For each Phase 1 gig in the catalog:

4a. Download the Thumbnail

Click the "PNG" button on the gig card. This downloads a 1280x769px thumbnail — the exact size Fiverr recommends.

4b. Copy the Title

Click "Copy Title". Paste it into Fiverr's title field. The character count is shown — make sure it's under 80.

4c. Copy the Description

Click "Copy Desc". Paste into Fiverr's description field. It's pre-formatted with bullet points and cross-sell callouts.

4d. Copy the Tags

Click "Copy Tags". Paste into Fiverr's search tags. Exactly 5 tags per gig (Fiverr's limit).

4e. Set Pricing

Match the 3-tier pricing from the catalog:

Tier Purpose Strategy
Basic Gets the click Low enough to attract browsers. This is your foot-in-the-door.
Standard Where money is made Marked as "POPULAR" — most buyers choose this. Best value proposition.
Premium Anchor price Makes Standard look reasonable. Some buyers actually pick this too.
Critical: Phase 1 First Only create Phase 1 gigs initially. Bad gigs drag down good gigs in Fiverr's algorithm. Wait until you have 5+ reviews before adding Phase 2 gigs. This is not optional — it's how the algorithm works.

5 Post-Launch Optimization

Follow this timeline after creating your gigs:

When Action Why
Day 1 Create Phase 1 gigs Fiverr boosts new gigs in first 2 weeks
Daily Respond to ALL messages within 1 hour Response time is a top algorithm signal
Daily Send 10 buyer requests Fastest way to get first orders
Week 2 Offer launch pricing to first 5 buyers 5 reviews = algorithm visibility threshold
Week 3-4 Add gig video (60-90 seconds) +30% impressions, 2x more orders
Week 6+ Launch Phase 2 gigs You have reviews — now add premium upsells
Level 2+ Launch Phase 3 gigs Expand based on what's selling

Fiverr Algorithm — What Actually Matters

Based on research across thousands of top-performing gigs:

DO (Algorithm Boosters)
  • Respond to every message within 1 hour
  • Use combo-niche keywords (2 skills combined = less competition)
  • Put your face in thumbnails (25-40% higher CTR)
  • Use dark backgrounds with bright accent colors
  • Keep 3-4 gigs in the same category (traffic sharing)
  • Get 5 reviews as fast as possible (visibility threshold)
  • Add gig FAQ section (converts browsers to buyers)
  • Cross-sell in every gig description
DON'T (Algorithm Killers)
  • Don't create gigs you can't deliver well — bad reviews kill everything
  • Don't scatter across unrelated categories
  • Don't compete on generic keywords ("website developer" = 100K+ gigs)
  • Don't set prices too low — it signals low quality
  • Don't ignore messages for hours — response time is tracked
  • Don't launch all gigs at once — phase them

Manual Mode (Without Claude Code)

If you don't have Claude Code, you can still use this tool manually:

  1. Clone the repo: git clone https://github.com/waseemnasir2k26/fiverr-gig-optimizer.git
  2. Copy config-example.json to gig-config.json
  3. Edit gig-config.json with your gig details (follow the structure exactly)
  4. Place your photo in the repo folder
  5. Run: python build-catalog.py
  6. Open fiverr-catalog.html in your browser
Config Structure Each gig in the config needs: id, phase, title, category, 5 tags, description (under 1200 chars), competition level, cross-sell text, 3-tier pricing with deliverables, and thumbnail settings (colors, headline, subtitle, badge, tool pills). See config-example.json in the repo for the full format.

Frequently Asked Questions

How many gigs should I create?

Start with 3-4 gigs in Phase 1, all in the same Fiverr category. This concentrates your traffic signals. Add more only after getting 5+ reviews on existing gigs.

What if I only have 1-2 skills?

You can still create 3-4 gigs by varying the scope: a basic version, a premium version, a bundle, and a specific use-case version. The optimizer handles this.

How long until I see results?

Fiverr boosts new gigs for 2 weeks. If you respond fast and get 5 reviews quickly (offer launch pricing), most sellers see consistent orders by week 4-6.

Can I edit the catalog after generating it?

Yes! Edit gig-config.json and re-run python build-catalog.py. Or run /fiverr-optimize again in Claude Code to regenerate from scratch.

Does this cost anything?

The tool is 100% free and open source. You only need a Claude Code account (free tier works) and a Fiverr account.

Can I use this for Upwork too?

The strategy applies to any freelance platform. The thumbnails and HTML catalog are Fiverr-specific (1280x769px), but the titles, descriptions, and pricing strategy work anywhere.
